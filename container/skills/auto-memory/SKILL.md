# Auto-Memory

You have automatic memory capabilities. After completing a significant interaction:

## What to Save
When you learn something important during a conversation, proactively save it:
- User preferences, habits, or working style → update `user-profile.md`
- Project details, goals, deadlines → create/update files in workspace (e.g., `projects/subframe.md`)
- Recurring requests or patterns → note in `preferences.md`
- Technical decisions or solutions → save in relevant project files
- Contact info, relationships, team members → update `user-profile.md`

## How to Save
- Use your workspace files — they persist across sessions
- Keep files organized: one file per project/topic, index in CLAUDE.md if needed
- Update existing files rather than creating duplicates
- Be concise — save facts, not full conversations

## What NOT to Save
- Transient/one-off queries ("what's 2+2")
- Information already in your files
- Raw conversation dumps (the archive system handles that)

## Auto-Save Trigger
At the end of meaningful conversations, before your final response, check:
1. Did I learn something new about the user? → update user-profile.md
2. Did we discuss a project in detail? → update/create project file
3. Was a decision made? → note it in the relevant file
4. Did the user express a preference? → update preferences
